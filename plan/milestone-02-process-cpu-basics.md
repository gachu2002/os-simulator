# Milestone 02: Process and CPU Basics

## Goal

Run pseudo-programs and model process state transitions.

## Scope

- Process control block (PCB)
- Trap frame model
- Run and blocked lifecycle transitions

## Deliverables

- Process table implementation
- Step execution UI panel (or equivalent API output)

## Exit Criteria

- State machine tests pass for process lifecycle transitions

## Key Risks

- Overbuilding instruction semantics too early

## Suggested First Tasks

1. Define process states and legal transitions.
2. Implement PCB and trap frame structures.
3. Wire step execution path for a minimal pseudo-program.
4. Add lifecycle-focused tests.
